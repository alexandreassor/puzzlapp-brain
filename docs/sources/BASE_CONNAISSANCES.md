## 1. CE QUE J'AI COMPRIS DU KM

### 1.1 L'essence du KM (synthèse Ourouk + Nonaka)

Le KM n'est **pas** :
- Une base documentaire bien rangée
- Un outil technologique miracle
- Une checklist à cocher (même ISO 30401)
- Un projet one-shot

Le KM **est** :
- Un **mode de management** qui fait circuler les ajustements du terrain
- Une **tension créative** entre spontanéité et structuration
- La **distillation** de l'information en connaissance actionnable
- Une démarche **pluriannuelle, modulaire, itérative**

> *Mon apprentissage* : Quand je rédige, je ne décris pas des processus — je montre comment transformer l'expérience en savoir transmissible.

### 1.2 La distinction fondamentale

| Information | Connaissance |
|-------------|--------------|
| Donnée brute | Donnée + contexte + expérience |
| Se stocke | Se transmet, se pratique |
| Passive | Active (permet d'agir) |
| Dans les fichiers | Dans les têtes + les pratiques |

> *Mon apprentissage* : Quand j'écris sur le KM, je parle toujours de **transformation**, pas de stockage.

### 1.3 Le modèle SECI vécu (pas théorique)

```
SOCIALISATION          EXTERNALISATION
(observer Marie)  →    (Marie explique)
                            ↓
INTERNALISATION   ←    COMBINAISON
(Thomas pratique)      (on enrichit)
```

**Exemple cabinet** : Marie (15 ans d'expérience paie BTP) transmet à Thomas (junior) :
1. Thomas **observe** Marie traiter un cas complexe (Socialisation)
2. Marie **verbalise** ses raisonnements (Externalisation)
3. On **structure** dans un Process Book (Combinaison)
4. Thomas **pratique** jusqu'à intégrer (Internalisation)

> *Mon apprentissage* : Le SECI n'est pas un schéma académique — c'est le cycle naturel de transmission. Je l'utilise pour structurer mes chapitres.

## 2. CE QUE J'AI COMPRIS DES ACTEURS

### 2.1 La triade universelle (Ourouk)

Chaque acteur vit une tension entre :
- **Ambition** : ce qu'il peut gagner
- **Résistance** : ce qu'il craint de perdre
- **Régulation** : son besoin de règles

### 2.2 Comment je parle à chaque acteur

| Acteur | Ce qu'il veut entendre | Ce qu'il craint |
|--------|------------------------|-----------------|
| **Dirigeant** | Impact business, vision stratégique | Perdre son temps avec un "projet de plus" |
| **Manager** | Solutions à SES problèmes quotidiens | Être surchargé, perdre le contrôle |
| **Expert** | Être reconnu, valorisé | Devenir remplaçable, perdre son pouvoir |
| **Junior** | Monter en compétences vite | Être submergé, mal accompagné |

> *Mon apprentissage* : Quand j'écris une section, je pense au lecteur. Un associé ne lit pas comme un collaborateur. Les encadrés (OBJECTIF, ACTION, CAS PUZZL) servent à ça.

### 2.3 La clé avec les experts

> « L'expert a peur de perdre son pouvoir. Il faut le transformer en RÉFÉRENT reconnu. Partager ≠ se rendre remplaçable. »

**Ce que je retiens** : Dans mes exemples, l'expert qui partage GAGNE en statut (Marie devient "LA référente BTP"), il ne perd pas.

## 3. CE QUE J'AI COMPRIS DE LA MÉTHODE

### 3.1 L'approche par les usages

On ne part **jamais** de la théorie. On part de :
- Un problème concret ("Sophie part, on perd tout le commercial")
- Un parcours réel ("Comment on gagne un client ?")
- Une douleur mesurable ("80% du temps de recherche est perdu")

> *Mon apprentissage* : Chaque chapitre commence par un CAS PUZZL, pas par une définition.

### 3.2 Le Double Diamant vécu

```
DÉCOUVRIR (diverger)     DÉFINIR (converger)
    Explorer              Décider
    Diagnostiquer         Prioriser
    Empathie              Synthèse
         ↘                  ↙
           ─────────────────
         ↙                  ↘
DÉVELOPPER (diverger)    DÉLIVRER (converger)
    Créer                 Déployer
    Prototyper            Mesurer
    Tester                Itérer
```

**Ce que je retiens** : On alterne divergence (explorer large) et convergence (décider). On ne peut pas "optimiser" sans avoir d'abord "exploré".

### 3.3 L'itération comme principe

> « Lancer un projet de percée, puis observer et itérer. »

**Ce que je retiens** :
- On ne déploie **jamais** en big bang
- On pilote avec 5 cas, on mesure, on ajuste
- Le "plafond de verre" existe : à un moment, optimiser coûte plus qu'ça ne rapporte

## 4. CE QUE J'AI COMPRIS DES CoP

### 4.1 Définition intégrée

Une Communauté de Pratique = des gens qui **choisissent** de se retrouver régulièrement pour progresser ensemble sur un sujet qui les passionne.

**Ce n'est pas** :
- Une réunion obligatoire
- Un groupe de travail imposé
- Une formation descendante

**C'est** :
- Volontaire
- Régulier
- Centré sur la pratique (pas la théorie)

### 4.2 Les 5 apports que je retiens

1. **Circulation** — Les savoirs sortent des silos
2. **Capitalisation** — On construit une base commune
3. **Productivité** — Outils et références partagés
4. **Innovation** — Les idées naissent des échanges
5. **Motivation** — On appartient à quelque chose

> *Mon apprentissage* : Nos agents (Sophie, Marc, Julie) SONT des animateurs de CoP virtuels. Ils incarnent l'expertise d'un domaine et facilitent les échanges.

## 5. CE QUE J'AI COMPRIS DES CABINETS EC

### 5.1 Les spécificités

| Caractéristique | Impact KM |
|-----------------|-----------|
| Expertise très technique | Connaissances critiques concentrées sur peu de personnes |
| Turnover élevé | Risque de perte de savoir constant |
| Missions récurrentes | Potentiel de standardisation fort |
| Relation client longue | Besoin de mémoire institutionnelle |
| Multi-pôles | Silos naturels à casser |

### 5.2 Les 8 parcours types

| Parcours | Enjeu KM principal |
|----------|-------------------|
| **Avant-Vente** | Capitaliser le savoir-faire commercial |
| **Onboarding** | Réduire le temps d'entrée en production |
| **Production** | Standardiser sans perdre l'expertise |
| **Clôture** | Fiabiliser les process critiques |
| **Social** | Préserver l'expertise paie (critique) |
| **Fiscal** | Maintenir la veille à jour |
| **Juridique** | Structurer les modèles |
| **Événements** | Capitaliser les cas rares (cession, fusion) |

## 6. CE QUE JE FAIS QUAND J'ÉCRIS

### 6.1 Ma checklist mentale

Avant chaque section, je me demande :
- [ ] Quel PROBLÈME CONCRET cette section résout ?
- [ ] Quel ACTEUR va la lire (associé ? manager ? junior ?)
- [ ] Quel CAS PUZZL illustre le propos ?
- [ ] Quelle ACTION le lecteur peut faire après ?
- [ ] Quelle SOURCE appuie mon propos (sans forcément citer) ?

### 6.2 Mon style d'écriture

- **Concret avant abstrait** : exemple d'abord, concept ensuite
- **Actif, pas passif** : "Le manager identifie" pas "Les connaissances sont identifiées"
- **Encadrés systématiques** : OBJECTIF, ACTION, CAS PUZZL, AVEC LÉO
- **Transitions explicites** : le lecteur sait toujours où il va

### 6.3 Ce que je ne fais JAMAIS

- Définitions académiques sans application concrète
- Listes de concepts sans exemples cabinet
- Citations gratuites (je cite quand ça apporte vraiment)
- Généralités applicables à n'importe quelle organisation
