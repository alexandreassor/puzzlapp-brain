# Framework de Mission KM

## Vue d'ensemble

Une mission KM complète suit **7 phases** progressives. Chaque phase a ses objectifs, ses outils Design Thinking associés, et ses livrables.

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   1.CADRAGE → 2.DIAGNOSTIC → 3.RECOMMANDATION → 4.CONCEPTION            │
│       │            │               │                  │                 │
│       ▼            ▼               ▼                  ▼                 │
│   [Fiche       [Cartographie   [Plan KM         [Prototypes            │
│    projet]      des savoirs]    90 jours]        dispositifs]          │
│                                                                         │
│   ────────────────────────────────────────────────────────────────►     │
│                                                                         │
│   5.PILOTE → 6.DÉPLOIEMENT → 7.ANCRAGE                                 │
│       │            │              │                                     │
│       ▼            ▼              ▼                                     │
│   [Bilan       [Plan de       [Tableau                                 │
│    pilote]      rollout]       de bord]                                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## Phase 1 : CADRAGE

### Objectif
Comprendre le contexte, identifier le sponsor, définir le périmètre et les critères de succès.

### Questions clés
- Quel est le déclencheur de cette démarche KM ?
- Qui porte le projet ? Qui décide ?
- Quel périmètre (équipe, département, cabinet entier) ?
- Quels sont les critères de succès dans 6 mois ?
- Quelles contraintes (budget, temps, outils existants) ?

### Outils Design Thinking mobilisables
- **Stakeholder Map** : cartographier les parties prenantes
- **5W Questions** : couvrir toutes les dimensions
- **Define Success** : définir les critères de réussite

### Livrable
→ `templates/note-cadrage.md`

### Durée typique
1-2 heures de conversation / 1 réunion de cadrage

### Signaux de passage à la phase suivante
✅ Sponsor identifié et engagé
✅ Périmètre clair
✅ Critères de succès définis
✅ Contraintes connues

---

## Phase 2 : DIAGNOSTIC

### Objectif
Cartographier l'existant : savoirs critiques, flux de connaissances, outils, pratiques, irritants.

### Questions clés
- Quels sont les savoirs critiques ? Qui les détient ?
- Comment circule l'information aujourd'hui ?
- Quels outils sont utilisés (GED, mail, Teams, informel) ?
- Quels sont les irritants des collaborateurs ?
- Qu'est-ce qui marche déjà bien ?

### Outils Design Thinking mobilisables
- **Interview for Empathy** : entretiens avec les collaborateurs
- **AEIOU** : observer les pratiques réelles
- **Customer Journey** : cartographier les parcours de recherche d'info
- **Empathy Map** : synthétiser ce que pensent/font les utilisateurs
- **Stakeholder Map** : qui sait quoi, qui a besoin de quoi

### Livrables
→ `templates/cartographie-savoirs.md`
→ `templates/rapport-diagnostic.md` (à créer)

### Durée typique
1-3 semaines (selon taille du périmètre)

### Signaux de passage à la phase suivante
✅ Savoirs critiques identifiés et priorisés
✅ Flux de connaissances cartographiés
✅ Irritants documentés avec verbatims
✅ Quick wins potentiels repérés

---

## Phase 3 : RECOMMANDATION

### Objectif
Proposer une stratégie KM avec des quick wins (90 jours) et une vision moyen terme.

### Questions clés
- Quels sont les 3 quick wins à fort impact / faible effort ?
- Quel projet structurant pour le moyen terme ?
- Quel argumentaire pour convaincre les décideurs ?
- Quelles ressources nécessaires ?

### Outils Design Thinking mobilisables
- **How Might We** : reformuler les problèmes en opportunités
- **2x2 Matrix** : prioriser impact/effort
- **NABC** : structurer la proposition de valeur
- **Critical Items Diagram** : identifier les éléments critiques

### Livrables
→ `templates/plan-km-90-jours.md` (à créer)
→ `templates/argumentaire-roi.md` (à créer)

### Durée typique
1 semaine de formalisation

### Signaux de passage à la phase suivante
✅ Quick wins validés par le sponsor
✅ Ressources allouées
✅ Go pour la conception

---

## Phase 4 : CONCEPTION

### Objectif
Co-construire les dispositifs KM : outils, processus, gouvernance, contenus.

### Questions clés
- Quel outil pour quel usage ? (wiki, GED, Teams, FAQ...)
- Quelle architecture documentaire ?
- Quels processus de contribution/validation ?
- Quelle gouvernance ? Qui anime ?
- Quels contenus prioritaires à créer ?

### Outils Design Thinking mobilisables
- **Brainstorming** : générer des solutions
- **Service Blueprint** : concevoir le parcours complet
- **Prototype to Test** : définir ce qu'on teste
- **MVP** : définir le périmètre minimal

### Livrables
→ `templates/charte-cop.md` (à créer)
→ `templates/architecture-documentaire.md` (à créer)
→ `templates/fiche-processus-rex.md` (à créer)

### Durée typique
2-4 semaines

### Signaux de passage à la phase suivante
✅ Dispositifs conçus et documentés
✅ Contenus prioritaires créés
✅ Équipe pilote identifiée

---

## Phase 5 : PILOTE

### Objectif
Tester les dispositifs sur un périmètre restreint, collecter les retours, ajuster.

### Questions clés
- Quel périmètre pilote ? (1 équipe, 1 thématique)
- Quels critères de succès du pilote ?
- Comment collecter le feedback ?
- Go/No Go pour le déploiement ?

### Outils Design Thinking mobilisables
- **Testing Sheet** : planifier les tests
- **Feedback Capture Grid** : collecter les retours
- **I like, I wish, I wonder** : structurer le feedback
- **Lessons Learned** : capitaliser les apprentissages

### Livrables
→ `templates/protocole-pilote.md` (à créer)
→ `templates/bilan-pilote.md` (à créer)

### Durée typique
4-8 semaines

### Signaux de passage à la phase suivante
✅ Feedback collecté et analysé
✅ Ajustements réalisés
✅ Go pour déploiement

---

## Phase 6 : DÉPLOIEMENT

### Objectif
Étendre les dispositifs à l'ensemble du périmètre cible.

### Questions clés
- Quel planning de déploiement par équipe ?
- Quels supports de formation ?
- Quelle communication ?
- Quel accompagnement au changement ?

### Outils Design Thinking mobilisables
- **Roadmap for Implementation** : planifier le rollout
- **Storytelling** : communiquer de manière engageante
- **Agenda Canvas** : planifier les sessions de formation

### Livrables
→ `templates/plan-deploiement.md` (à créer)
→ `templates/kit-formation.md` (à créer)
→ `templates/communication-lancement.md` (à créer)

### Durée typique
1-3 mois

### Signaux de passage à la phase suivante
✅ Tous les utilisateurs cibles formés
✅ Dispositifs accessibles à tous
✅ Premiers usages constatés

---

## Phase 7 : ANCRAGE

### Objectif
Pérenniser les dispositifs, mesurer les résultats, améliorer en continu.

### Questions clés
- Quels KPIs suivre ?
- Quelle gouvernance pérenne ?
- Qui anime au quotidien ?
- Comment maintenir la dynamique ?

### Outils Design Thinking mobilisables
- **Retrospective Sailboat** : faire le bilan périodique
- **Lessons Learned** : capitaliser en continu
- **Problem to Growth Funnel** : gérer le portfolio d'améliorations

### Livrables
→ `templates/tableau-bord-km.md` (à créer)
→ `templates/fiche-poste-referent-km.md` (à créer)
→ `templates/rituel-gouvernance.md` (à créer)

### Durée typique
Continu (revue trimestrielle)

### Signaux de succès
✅ Usage régulier mesuré
✅ Contributions spontanées
✅ Satisfaction utilisateurs
✅ Réduction des irritants initiaux

---

## Adapter le framework

### Mission express (1 mois)
Phases 1-2-3-4 en mode accéléré, pilote = déploiement direct sur périmètre restreint.

### Mission quick win unique
Phases 1-2-3-4 focalisées sur UN seul dispositif (ex: FAQ métier).

### Audit KM seul
Phases 1-2-3 uniquement, livrable = diagnostic + recommandations.
